==================================================================
LENOVO TAB PLUS (TB351FU) - ORANGEFOX 14.1 PORTING CHRONICLES
Chipset: Helio G99 (MT6789) | OS: ZUI 17 (SDK 36 / Android 14+)
==================================================================

1. THE ARCHITECTURE (GKI Header v4)
-----------------------------------
The device uses a modern Generic Kernel Image (GKI) structure. 
Recovery is not a separate partition; it lives as Fragment 1 
inside the 'vendor_boot' partition.

2. THE SECURITY BARRIERS
------------------------
* RECOVERY BOOTLOOP: Triggered if the bootloader detects a mismatch 
  in Fragment 0 (Platform).
* WATCHDOG REBOOT: A hardware timer (watchdogd) reboots the device 
  at exactly 9 seconds if the recovery UI fails to send a heartbeat.
* PRODUCTION LOCK: Stock 'adbd' refuses root access if the 
  'ro.debuggable' bit is not hard-patched in the binary.

3. THE SOLUTIONS (STABILIZATION SUITE)
--------------------------------------
* FRAGMENT FUSION: We kept Fragment 0 100% stock to satisfy the 
  bootloader, and injected OrangeFox into Fragment 1.
* PIXEL FORMAT: Forced 'BGRA_8888' to match the G99 GPU requirements.
* STRIDE FIX: Forced 'RECOVERY_GRAPHICS_FORCE_USE_LINELENGTH' to fix 
  the partial/distorted UI rendering.
* WATCHDOG KILL: Injected a background loop into the init script 
  to 'pkill -9 watchdogd' every 5 seconds.
* HARDWARE SYNC: Explicitly started 'health-hal' and 'mtk_plpath_utils' 
  to enable the Goodix/Novatek touchscreen drivers.

4. GENERATION PROCESS
---------------------
The working image is generated by compiling the OrangeFox 14.1 
source against SDK 36 headers, then surgically swapping the 
recovery fragment inside the stock Lenovo 'vendor_boot.img' 
container. 

* SIZE CONSTRAINTS: The partition size is exactly 67,108,864 bytes (64 MiB). 
  Internal recovery size is not a constraint as long as the total image 
  remains under this limit.
* PACKAGING: We use a custom mkbootimg configuration to align the 
  platform and recovery fragments.
